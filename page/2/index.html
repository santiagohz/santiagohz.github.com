几点Java笔记：

  • 构造函数和方法调用：http://www.cnblogs.com/zhangxindeshui/archive/2010/08/17/1801506.html
  • String不是基本数据类型。具体使用方法可以参考这篇Blog。
  • Java对大小写敏感，类、方法和变量命名的时候需要注意。基本命名规范如下：
    • 包名：多单词组成是全部为小写字母。例如package com.santigaohz.test1;。
    • 类名和接口名：单词首字母都需要大写，例如public class MainActivity。
    • 变量名和函数名：驼峰命名，例如protected void onCreate(Bundle savedInstanceState)
    • 常量名：所有字母都大写。多个单词组成时，用下划线（“_”）连接。如MAX_VALUE

2013年的最后一月

还未觉察，2013年最后一月就这样到来。

微博上有人每个月第一天都会发条微博：“1月开始了！”，“2月开始了”，。。。直到今天写到：“12月开始了！”。所以，时间也不是无声无息，有些敏感的人会察觉到它的流逝。这个月就从这样一个平淡的周天开始，而明天，我就要正式开始下个工作。不紧张，不忐忑，只觉得内心很平静。最后一个月，让自己充实起来，不虚度这珍贵的每一天。

这几天在看Java，看到书里的内容，我依稀回忆起大三在管理楼上Java课的场景。只可惜那时还没找到以后的兴趣所在，所以就没有用心去学习。现在看来好惋惜。现在只能努力去理解对于我而言稍显复杂的对象声明，类方法，实例化，方法调用。不过现在的我很喜欢这样的过程，学习和成长的道路总不会一路平坦，那些让你现在困难的事情只会给以后的你增加分量。如果能回到当初，我会告诉那时的我现在的感悟。

上周买的花好像有点问题，叶子发黄了。可能这个我忘记名字的花不太喜欢呆在光线昏暗的室内。我决定把它放在阳台上呆几天去透透气，希望它能快快恢复。
花儿啊，你可知道，在一个新的环境里，最重要的就是要学会适应！我在努力，你也要加油。

Ctrl+C不能用的问题

重装系统之后，发现Ctrl+C的快捷键用不了了。很是不方便。今天搜索一下，发现有人提的到输入法的热键设置可能会导致这个问题。
故而看了一下搜狗输入法的快捷键设置，原来真的把切换搜狗输入法的热键设置成了Ctrl+C（明明设置的是Ctrl+Space，但是一直显示的却是Ctrl+C。不知神马原因）。
因此取消了热键切换，Ctrl+C可以用了，瞬间感到世界真美好~

装系统的奇葩经历

前天准备学习Android Coding，对比了一下现在的开发平台，决定还是从头开始使用Android Studio 官网地址。结果装上之后，果然如大家所说，这个预览版本对机器性能的要求太高，搞得我这个老E420巨慢无比。后来考虑升级笔记本内存，计划买一条4G的内存，组成6G双通道。然后查了一下原有的Win7系统是32位，最多只能支持4G内存的上限。所以决定再把32位的操作系统升级到64位的Win8，以解决内存上限的问题。然后查找升级攻略，发现因为位数不一样，只能用系统盘来安装，然后跑到理工大学附近买了一张系统盘。升级到Win8之后发现需要激活系统，研究了一下现在的Win8还没有破解版本，准备去买一个正版系统。在准备去下单的时候，看到网上有说可以通过英国电话激活。然后研究了一下，结果真的激活成功。好吧，200多元的购买基金就暂时先存下，之后到其他平台上去支持吧。
