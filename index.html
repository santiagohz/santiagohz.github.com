未来半年的几个目标

今天和海信的hr联系了一下，入职的流程还在走。貌似部门在调整，目前还得不到准确的入职时间，看来还要再等一些时间。国企的效率真的是太低下，以这样的速度去和互联网那些野蛮种族战斗，后果确实不容乐观。

后来追问了hr一句，问有没有可能部门调整offer审批不通过？得到的答复是有这种可能性。好吧，如果最后真就这样，那就在青岛住上半年吧。用半年的时间缓一缓自己膨胀的速度，也不失为一件好事。不管怎样，今天设定几个目标，从明天开始监督自己去实现。

1. 专业学习。

之前一直在做产品，虽然接触过一个产品生命周期的每一个环节，但是对于产品本职之外的工作掌握的还是不够深入。比如Coding和PS。虽然不一定能在目前的工作上用的上，但是出于一直以来对技术的敬畏，我还是想自己掌握以下两个技能。

Coding: 虽然iOS更简洁，但是考虑Android系统的开放性。Coding还是从Android入手吧。初期目标先能做一些简单的演示demo，以便之后能够更好地阐述自己的设计思路。

PS： 我个人认为现在的扁平化趋势其实降低了UI设计的门槛，遵循一些平台的设计规范可以更方便地做出效果。初期目标先配合自己的coding学习，做一些基础的ui页面设计。

学习新的技术不是一个简单的事情，中间肯定会产生放弃的念头。就以此文章为证，加油吧郝哲，Never give up!

2. 写自己的Blog。

想想过去的两年多时间，自己经历过了不少事情，可惜都没有总结整理下来。很多经验就此过去，只能成为经历，少了沉淀和感悟。所以从当下开始，多把自己的经历和想法记录下来。先为自己的Blog写几个Tag，也算是之后学习知识的分类。

产品：之前在Wordpress上写的Tag名字叫“我的产品观”，就是记录一下自己的对于产品工作的一些经验和看法。可惜Wordpress的后台被我拆了，不知道之前的文章还能否再恢复。在现在的后台上我简化为“产品”两个字吧，不管是感悟还是对于新兴事物的认识，都在这个Tag下多写写。

投资：前两天看Bitcoin的疯狂表现，也叹息自己不懂抓住机会。其实作为互联网的一份子，对技术更快的认识、接纳以及更快的消息获取速度都是自己的优势。只是之前不懂得如何整合不同信息，并将以联系和利用。以后学习一些美股投资知识，利用自己在互联网的知识和信息，尝试关注和投资一些中概股。赚钱的任务很艰巨，就从力所能及的点滴开始吧。

健身：健康无价。尤其是刚经历了大爷的过世，更感受到生命的脆弱和无力。来青岛也是有健康这方面的考虑。除了平常的足球，还是需要再把跑步捡起来。昨天出去跑了20分钟，能感受到身体机能退化严重。以后如果没有异常，每天晚上抽40分钟出去跑一下。希望能找回当初在东大校园夜跑的自己。另外，每天的作息需要再规律一些，控制在12点之前睡觉。

美食：算算迄今已经做过4个菜了，成果还算不小，只是水平还是差距太大，当然也表示进步的空间很大。明天早上尝试第4个新菜，醋溜土豆丝。喜欢下厨房的Slogan，唯有美食与爱不可辜负。

随感：唯有美食与爱不可辜负，美食可以自己解决，但是爱却要两个人。只是未来的你，如果你不愿早日出现，我希望你呆在正确的时间和地点，等我来找你。这个Tag下将会记录这一路寻觅的点点滴滴，等我有一天，将它更改为爱情。

当然还有很多新的知识需要学习，比如要再美化一下自己的Hexo模板，学习一下Git的使用（Coding的时候应该可以用到），学习养花，学习手工制作，等等。总之需要学习的东西太多，定下目标，一步一步慢慢来。

人生急促又缓慢，就从这个不断学习的过程中去探索人生的意义吧。

今天的优化

晚上看了一个电影白宫陷落,一般水平的电影。场面还可以，就是情节设计上有不少脑残之处。实在忍不住吐槽男主能不能在生死攸关的时候不那么毫无顾忌、慢慢腾腾地打电话？！还有女主能不能紧张一点，这是白宫被攻陷了，不是一个监狱被攻陷了~啊亲！！以及男主的女儿，表演的太过了，明显超出年龄层次的表情~有木有！！！

吐槽完毕，回归正题，今天对blog进行了两点优化：

补充About页面内容

具体方式是：

  1. 在/source文件夹中新建一个About文件夹。注意这个文件夹名字应该和主_config中的About命名保持大小写一致。
  2. 在该文件中新建一index命名的.md文件（直接从中/source/_post文件夹中copy一个.md文件过来，把名字修改一下即可）。
  3. 把自己的描述用Markdown格式书写一下。普通青年可以介绍下自己的基本信息，兴趣爱好，联系方式等。文艺青年可以写首诗。2B青年可以随意发挥。
    我把这个blog标题命名为“矛盾论”，不要问我原因，你猜。

支持中文

昨天的blog标题显示中文乱码。后来查了一下原来是没有对配置文件使用UTF-8编码的问题。解决方案很简单：使用notepad++打开_config文件，选择Encoding > Encode in UTF-8，然后保存即可。
此方式对于所有中文显示乱码都适用，只要找到该乱码文字所在的文件，然后打开后用UTF-8格式保存一下既能解决。

域名定向

具体方式就是把Godaddy的域名重新进行NDS解析（可以通过中间的DNSPod服务），然后把解析的域名定向到当前的blog。具体操作步骤请见参考.不过我设置完之后到现在也没有定向成功，不确定是我没有设置正确还是DNS服务解析需要时间（根据DNSPod的说法，重新解析定向之后最多需要72个小时才能生效。）所以现在还要等一等，看后面的结果如何。

以上。要去睡觉了，明天周六，have a good day tommorrow.

Welcome to the new world

搞了两天终于搭起了这个简单的blog。对于一个非技术来说真心不太容易。小小感慨一下。

心路历程

最开始在wordpress上搭了一个blog：zhezhe.me，不过购买的空间国内访问速度太慢。近乎于废弃。中间又因为升级wordpress后台，而把ftp上的配置数据给删除了。（不知还能否再找回来之前写的一些文章，之后再尝试恢复一下。）总之，用的不太舒服，想优化优化。

后来又接触到Markdown。总是好奇心作祟，又忍不住顺藤摸瓜，找到了Github，并了解了程序员专用的git。对于一个手工崇拜者来说，找到这样一个简单可以尝试的新手艺（我把编程视为一门手艺，这是称赞用语），是一件很高兴的事情。就这样，找了一下资料开始学习在github上搭建blog。

收获

期间学习了一些新的知识：

  • Git
  • Github
  • jeklly
  • hexo(当前这个blog采用的框架）
  • Node.js
  • Markdown

虽然仅仅接触到一些皮毛，但是了解到新的东西还是很兴奋。中间学习搭建过程，搜索了一些独立blog。看到很多人都有一些自己的坚持，还是觉得很佩服。尤其是对于Hexo的作者tommy351，由衷敬佩这些坚持的人。 世界因为这些可爱的人而精彩

后续计划

今天搭建完，权当是记录一个流水账。后面几天计划陆续完善以下:

  • 优化blog模板样式
  • 把zhezhe.me的域名定向过来
  • 把自己在搭建过程中找到的资料和跳过的坑整理一下，放出来，以飨后来人
  • 学习Markdown，写一些文章
